# PBPK MCP v0.2.2

Suggested GitHub release title:

- `PBPK MCP v0.2.2: OECD dossier export, performance evidence, and release readiness`

Suggested short release summary:

- PBPK MCP `v0.2.2` strengthens the OECD-oriented qualification layer with structured dossier export, model-performance evidence support, parameter provenance export, and a release-readiness workflow for the live stack.

## Highlights

- adds `export_oecd_report` across supported backends
- adds `modelPerformance`, `parameterProvenance`, and `performanceEvidence` support in the qualification layer
- supports dossier-oriented R hooks such as `pbpk_parameter_table(...)` and `pbpk_performance_evidence(...)`
- adds a pre-release `release_readiness_check.py` workflow against the running stack
- expands architecture documentation with a logical architecture diagram and a runtime deployment diagram

## Included In This Release

### OECD-oriented reporting improvements

- structured OECD-style dossier export for supported loaded models
- export of normalized `profile`, `validation`, `oecdChecklist`, and `oecdChecklistScore`
- optional parameter provenance table export
- optional performance-evidence export when the model or sidecar provides it

### Documentation and architecture improvements

- clearer architecture documentation for the unified API plus dual-backend design
- explicit documentation of limitations, qualification boundaries, and conversion-only formats
- updated release and publication guidance

### Operational readiness improvements

- release-readiness check for health, discovery, validation, reporting, and result retrieval
- more honest qualification semantics around runtime-only or smoke-only evidence

## Limitations To Call Out Publicly

- Berkeley Madonna `.mmd` is still not directly executable
- `.pksim5` still requires export to `.pkml`
- population simulation remains `rxode2`-specific rather than generic across all backends
- model-performance and provenance sections may still be partially populated for many models
- dossier export does not by itself make a model regulatory-ready

## Suggested Release Notes Body

```markdown
PBPK MCP `v0.2.2` strengthens the OECD-oriented qualification layer while keeping the user-facing workflow unified across supported PBPK backends.

This release adds:

- `export_oecd_report` for structured dossier export
- richer qualification metadata for `modelPerformance` and `parameterProvenance`
- optional structured `performanceEvidence` export
- R model hooks for parameter-table and performance-evidence export
- a `release_readiness_check.py` workflow to verify the running local stack before publication

It also improves the documentation around architecture, limitations, and qualification boundaries.

Important limitations remain explicit:

- Berkeley Madonna `.mmd` is a conversion source, not a runtime backend
- `.pksim5` remains an export source rather than a direct runtime format
- runtime success and OECD-style metadata do not automatically imply regulatory qualification
```
