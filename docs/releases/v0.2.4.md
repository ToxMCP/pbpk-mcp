# PBPK MCP v0.2.4

Suggested GitHub release title:

- `PBPK MCP v0.2.4: qualification states and model manifest validation`

Suggested short release summary:

- PBPK MCP `v0.2.4` strengthens the OECD-oriented qualification workflow by adding explicit qualification-state labels and static manifest validation for supported PBPK model formats.

## Highlights

- adds `validate_model_manifest` for pre-load completeness checks on supported `.pkml` and MCP-ready `.R` models
- exposes `qualificationState` in validation responses and OECD report exports
- adds `scripts/validate_model_manifests.py` for release-gating and inventory-wide manifest checks
- tightens documentation around qualification semantics and manifest completeness limits

## Included In This Release

### Qualification-state layer

- reusable qualification-state derivation added to the bridge
- `qualificationState` surfaced through `load_simulation`, `validate_simulation_request`, and `export_oecd_report`
- qualification outcomes now distinguish exploratory, research-use, illustrative, and context-qualified model states more explicitly

### Static manifest validation

- new `validate_model_manifest` MCP tool
- new `model_manifest` validation layer for `.pkml` sidecars and MCP-ready `.R` model modules
- new `scripts/validate_model_manifests.py` batch validation script for release checks

### Documentation and release readiness

- README and architecture docs updated to explain manifest validation and qualification-state semantics
- publication checklist and release-readiness guidance updated to include manifest validation coverage
- regression coverage added for manifest-state classification and qualification-state propagation

## Limitations To Call Out Publicly

- manifest validation is a static completeness and qualification-signal check, not proof of scientific validity
- runtime success and dossier export still do not automatically imply regulatory qualification
- Berkeley Madonna `.mmd` remains a conversion source, not a runtime backend

## Suggested Release Notes Body

```markdown
PBPK MCP `v0.2.4` strengthens the OECD-oriented qualification workflow with two important additions:

- explicit `qualificationState` labels across validation and report surfaces
- static `validate_model_manifest` checks for supported `.pkml` and MCP-ready `.R` models

This release makes it easier to distinguish between exploratory, research-use, illustrative, and more strongly qualified model states before execution, while also improving release-gating and inventory-wide validation workflows.

It does not change the core dual-backend execution model:

- `ospsuite` remains the runtime for supported `.pkml` models
- `rxode2` remains a first-class runtime for MCP-ready `.R` PBPK models

Important limitations remain:

- manifest validation is a pre-load completeness check, not scientific proof
- runtime success and OECD-style dossier export do not automatically imply regulatory qualification
- Berkeley Madonna `.mmd` is still a conversion source rather than a direct execution backend
```
