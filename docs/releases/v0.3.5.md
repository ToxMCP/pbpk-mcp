# PBPK MCP v0.3.5

## Release Summary

- `PBPK MCP v0.3.5: stronger uncertainty semantics and deploy stability`

## Overview

PBPK MCP `v0.3.5` strengthens the evidence model and the operator path without changing the public workflow shape introduced in `v0.3.x`.

This release adds:

- typed external uncertainty handoff references for PBPK-side NGRA handoff objects
- stricter uncertainty semantics for quantified versus declared-only propagation evidence
- stronger predictive-evidence traceability checks against declared datasets, target outputs, and acceptance criteria
- deploy-path readiness stabilization for the patch-first local stack

It does **not**:

- change qualification scoring semantics
- move BER calculation or decision logic into PBPK MCP
- publish local/example model assets

## Added

- `uncertaintyRegisterReference.v1` for optional external uncertainty-register provenance
- `uncertaintySummary.semanticCoverage` counters such as `quantifiedRowCount` and `declaredOnlyRowCount`
- `scripts/wait_for_runtime_ready.py` for stable local deploy/readiness checks

## Changed

- `variability-propagation` rows are only treated as quantified propagation evidence when they actually carry quantitative outputs
- `uncertaintyHandoff.supports` now reflects whether typed PBPK-side uncertainty semantics and quantified uncertainty evidence are actually attached
- predictive-evidence traceability now warns when bundled rows do not match declared datasets, target outputs, or acceptance criteria
- `deploy_rxode2_stack.sh` now waits for stable `/health` and `/mcp/list_tools` responses before returning

## Why It Matters

PBPK MCP is now a better substrate for downstream NGRA orchestration because the uncertainty boundary is clearer, predictive support is harder to overstate accidentally, and local patch-first redeploys are less operationally brittle.

That matters because it improves:

- PBPK-side uncertainty traceability for downstream consumers
- evidence hygiene for predictive-support bundles
- stability of immediate post-deploy live verification
- confidence that the patch-first runtime has actually settled before smoke tests begin

## Verification

The release was verified with:

```bash
make runtime-contract-test
python3 -m unittest -v tests/test_model_discovery_live_stack.py tests/test_oecd_live_stack.py
python3 scripts/release_readiness_check.py --skip-unit-tests
```

The deploy-stability increment was also verified by redeploying through:

```bash
./scripts/deploy_rxode2_stack.sh
```

and confirming that it returned only after stable `/health` and `/mcp/list_tools` responses were available.
