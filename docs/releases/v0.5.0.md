# PBPK MCP v0.5.0

## Release Summary

- `PBPK MCP v0.5.0: dynamic physiological bounds, session persistence, and calibrated resource quotas`

## Overview

PBPK MCP `v0.5.0` introduces three major production-readiness improvements that close long-standing operational gaps:

1. **Dynamic, weight-scaled parameter bounds** — anatomical and physiological parameter mutations are now validated against allometrically-scaled min/max bounds derived from the simulation's body weight.
2. **Simulation persistence across container restarts** — the API automatically rehydrates previously-loaded simulations from a Redis-backed session registry on startup, eliminating the "Simulation not found" surprise after restarts.
3. **Calibrated population memory quotas** — the resource-quota estimator was updated from a placeholder (`0.5 MB/patient`) to a live-measured value (`0.25 MB/patient`) after rxode2 profiling at 100–2,000 subjects.

In addition, all development-only JWT secrets were lengthened to 32+ bytes, eliminating PyJWT `InsecureKeyLengthWarning` spam across the test suite and runtime logs.

## Added

- `ParameterBoundsRegistry` now supports allometric scaling metadata (`scale_reference_weight`, `scale_reference_value`, `scale_exponent`, `scale_tolerance_factor`)
- Cross-parameter consistency validator covering 13 organs (volumes vs. body weight, blood flows vs. cardiac output, hepatic clearance vs. blood flow)
- `run_parameter_consistency_check` MCP tool for on-demand simulation-wide physiological audits
- Session rehydration helper (`_rehydrate_sessions`) that replays snapshots on API startup
- Redis session backend configuration in `docker-compose.celery.yml`
- `scripts/population_load_test.py` for live rxode2 worker memory profiling
- Unit tests for startup rehydration (`tests/unit/test_startup_rehydration.py`)
- Live-measured regression tests for memory estimation (`tests/test_resource_quota.py`)

## Changed

- `memory_per_patient` in `ResourceQuotaValidator` changed from `0.5` (placeholder) to `0.25` (measured)
- `AUTH_DEV_SECRET` and all sibling test secrets lengthened to 32+ bytes
- `docker-compose.celery.yml` now defaults to `SESSION_BACKEND=redis` with `SESSION_REDIS_URL=redis://redis:6379/2`
- JSON-RPC transport now correctly passes `snapshot_store` to tool handlers

## Fixed

- `InsecureKeyLengthWarning` from PyJWT in all dev/test environments
- Simulation ephemerality after API container restarts
- Population jobs >200 subjects failing silently due to missing `snapshot_store` injection in JSON-RPC path

## Verification

The release state was checked with:

```bash
./.venv/bin/pytest tests/ -q
python3 scripts/population_load_test.py --sizes 100 500 1000 1500 2000
./.venv/bin/python scripts/check_release_metadata.py
./.venv/bin/python scripts/check_distribution_artifacts.py
```

Those checks confirmed:

- **285 passed, 1 skipped** across the full test suite
- Live rxode2 memory regression: `slope = 0.248 MB/patient`, `R² = 0.952`
- Estimator remains conservative above all observed peaks
- Release metadata alignment at `0.5.0`
- Healthy stack after API restart with zero manual simulation reloads

## Migration Notes

- No breaking API changes
- Existing clients benefit transparently from session persistence
- Operators should ensure Redis AOF persistence is enabled (already default in `docker-compose.celery.yml`)
